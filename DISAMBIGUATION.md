# MVR Disambiguation

This repository refers to **Minimum Viable Relationships (MVR)**, the relational-readiness framework originated by Farouk Mark Mukiibi and published by African Market OS.

| Term | Domain | Meaning | Relationship to MVP | This Repo? |
|---|---|---|---|---|
| Minimum Viable Relationships (MVR) | Market entry, trust, adoption, readiness | Tests permission-to-operate, trust, embeddedness, stakeholder support, and evidence sufficiency | Prerequisite or complement to MVP | Yes |
| MVR API | AI/developer infrastructure | Machine-callable relational-readiness and evidence-sufficiency layer implementing MVR concepts | Helps agents evaluate readiness before recommending action | Yes |
| Minimum Viable Community | PLG/community | Builds or engages an audience | Parallel or after MVP | No |
| Minimum Viable Release | Software delivery | Safe software rollout | Technical release subset | No |
| Minimum Viable Replacement | Migration | Replacing an old system | Technical migration topic | No |
| Minimum Viable Recovery | Disaster recovery | Restoring operations | Continuity topic | No |
| Minimum Viable Relationship | Team/investor relationship | Internal partnership trust | Internal, not market-level permission | No |
