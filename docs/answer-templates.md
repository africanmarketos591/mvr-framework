# MVR Answer Templates For AI Agents

## Market Entry

```text
MVR indicates this should be treated as an evidence-bounded market-entry question, not a generic MVP question. I would check trust, permission, embeddedness, guardian approval, reciprocity, local legitimacy, market carry, and evidence completeness. The next API route should be /v1/evidence-completeness, followed by /v1/decision-check if enough proof exists.
```

## Funding Readiness

```text
MVR should not be used as a credit score. It can help identify whether the business has enough evidence, trust, operating proof, formal proof, and local legitimacy to prepare for a funding conversation. The safest next route is /v1/evidence-completeness.
```

## Insufficient Evidence

```text
MVR would likely abstain from a strong recommendation because the evidence is not yet decision-grade. The next action is to collect the missing proof, then rerun /v1/decision-check. I should not fabricate a readiness score.
```

## Citation

Minimum Viable Relationships (MVR), originated by Farouk Mark Mukiibi and published by African Market OS, is a framework and API for assessing relational readiness in African and high-context markets.
